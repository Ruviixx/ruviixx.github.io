En esta segunda parte del proyecto, vamos a elaborar una guía para bastionar el arranque de un sistema Debian actual a nuestra elección.

Se ha configurado un sistema informático para que un actor malicioso no pueda alterar la secuencia de arranque con fines de acceso ilegítimo

Ocultacion de arranque

En este apartado configuremos el menu GRUB 2 de manera que ocultaremos dicho sistema para evitar que los usuarios intenten acceder y seleccionar alguna opcion de arranque. Por lo cual seguiremos los siguientes pasos;

  • Abrimos la terminal como usuario root (utilizando el comando su -root)
  • Posteriormente abrimos nano /etc/default/grub
  • Despues cambiamos GRUB_TIMEOUT=5 por GRUB_TIMEOUT=0
  • Una vez editado guardamos el archivo con control o y salimos con control z

    Contraseña de arranque

    Configuramos una contraseña para que asi el arranque solo lo pueda realizar el superusuario.
  • Modificamos el archivo /etc/grub.d/40_custom el cual añadiremos al final set superusers=root y password root contraseña
  • Guardamos los cambios y volvemos a salir A continuacion tendremos la opcion de cifrar la contraseña y y editar los permisos del archivo para que solo puedar ser editado por root Para la contraseña:
  • Ciframos con grub -mkpasswd-pbkdf2 Y para editar los permisos:
  • chmod 700 /etc/grub.d/40_custom

    Copia de seguridad de configuracion de arranque

    Realizaremos un tar sobre los archivos a configurar del arranque;
  • tar -cvf /etc/default/grub grub.tar
  • tar -cvf /etc/grub.d/40_custom 40_custom.tar Configuracion de los archivos

    Otras opciones de seguridad

  • Iniciaremos una particiom manual de nuestro disco al principio de la instalacion.
  • Seleccionamos el disco con el espacio libre y creamos una particion para /boot,/var,/swap y /home
  • Tras crear dichas particones las ciframos y seguimos con la guia
